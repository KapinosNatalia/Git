1.Как отказаться от изменений в файле (файлах) находящийся в рабочем каталоге (work directory)?
Не очень уверена... 
"git restore имя_файла"
Можно применить до того, как изменения добавлены в систему контроля версий командой "git add имя_файла"

2.Как вернуть файл из индекса в рабочий каталог?
После того, как изменения добавлены командой add, их можно вернуть командой
"git restore --staged имя_файла"

3.Как отменить ошибочный commit ?
git reset HEAD~

4.Как создать новую ветку и переключится в нее?
git branch название_ветки - создание ветки
git checkout -b название_ветки - переключение на нужную ветку

5.В чем отличение между fast-forward merge  and non fast-forfard merge?
fast-forward merge - это простой вариант мержа. Если мы, например, создали ветку branch1 от ветки main и больше никаких коммитов в ветке main не делали, и решили смержить ветку main и branch1, то будет выполнен этот вариант мержа. В этом случае новый коммит не выполняется, и ссылка ветки main переносится на последний коммит ветки branch1.
non fast-forward merge - более сложный вариант мержа, когда в ветке main были коммиты после того, как от нее была создана ветка branch1. В этом случае будет создан новый коммит, у которого будут два родительских коммита: из ветки main и из ветки branch1

6.Когда может возникнуть конфликт слияния веток, как  его разрешить?
Конфликт слияния может возникнуть, когда в двух ветках, которые мы хотим сливать были выполнены изменения в одном файле, в одной строке файла, или в одном из коммитов файл был удален, т.е. были выполнены изменения одного и того же объекта. 
В этом случае при выполнении команды merge гит покажет, где происходит конфликт, и нужно будет вручную указать, какие изменения нужно оставить.

7.Можно ли удалить ветку? Как? 
git branch -d название_ветки - удаляет ветку только если она уже была смержена с какой-то другой перед удалением
git branch -D название_ветки - удаляет ветку независимо от ее статуса
